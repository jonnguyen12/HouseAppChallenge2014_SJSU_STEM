Planning

-When coding a project, whether it is a small and simple game or an assignment involving the use of complex data structures, it is always best to hit the scratch paper before hitting the keyboard. Planning and designing your code is a very important step to a bug free and beautifully written program; I usually split at least one third of my dedicated time for a project into the design process itself.

-Suppose I want to make a simple game using the Scratch software. First I decide what kind of game I want to play, the characters (or should I say Sprites) involved, and how the game is won/lost. After some time thinking, I decide that the game would involve a Sprite that I use to move around, a flag of some kind that would win me the game if I touch it with my Sprite, and some obstacles in the middle.

Design

Now that we have the design, let’s start writing some Pseudocode. Writing good pseudocode is also very important. Real code is not that important, because with good pseudocode, one can easily translate it into a specific language like Java, C, etc. So now we write some pseudocode for each one of our objects.

My Object: After some thinking, I decided that my object would be moved by arrow keys, display a winning message when I touch the flag, a losing message when I touch an obstacle, and start out at the bottom left of the screen.

  • 1 place object at bottom left
  • 2 while game is not over
    • 1 if left arrow pressed
      • 1 move object left
    • 1 if left arrow pressed
      • 1 move object left
    • 2 if right arrow pressed
      • 3 if up arrow pressed
    • 1 move object up
      • 4 if down arrow pressed
    • 1 move object down
      • 5 if reached flag
    • 1 move object down
      • 5 if reached flag
    • 6 if touch obstacle
      • 1 display losing message

Obstacle: I want my obstacle to move with some basic physics behind it, velocity and acceleration. I also want the object to reverse its direction when touching the bound.

1 create and set my velocity and acceleration 2 place obstacle at desired position 3 while game is not over 1 add acceleration to velocity 2 move the object using velocity 3 if touching bound 1 set current velocity to negative 2 move the object using velocity

Flag: I want to spice things up a bit and have the flag move around randomly every 5 seconds instead of just staying still.

1 place obstacle at desired position 2 set i to 0 3 while game is not over 1 if i divides 5 evenly 1 move object to a random position on screen 2 increment i
